You’re seeing that error because **`doFilterInternal()` is `protected`** in `OncePerRequestFilter`, and your test class is in a **different package**, so it cannot call it directly. The method is defined this way in .

There are **three common solutions**. The best one for unit tests is **Option 1**.

# Option 1 (Recommended): Call `doFilter()` instead

`doFilter()` is **public** and internally calls `doFilterInternal()`.

So update your test to use `filter.doFilter(...)`.

### Fixed Test


# Option 2: Put the test in the same package

Move the test to:

```
src/test/java/com/uol/comp3011/coursework1/config/security/
```

Then you *can* call:

```java
filter.doFilterInternal(request, response, filterChain);
```

because **protected methods are visible within the same package**.

---

# Option 3 (Not Recommended): Use Reflection

```java
ReflectionTestUtils.invokeMethod(
        filter,
        "doFilterInternal",
        request,
        response,
        filterChain
);
```

Works but makes tests uglier and harder to maintain.

---

# Best Practice (What most Spring projects do)

Use:

```java
filter.doFilter(request, response, filterChain);
```

because it tests the **real filter behaviour**, not just the internal method.
